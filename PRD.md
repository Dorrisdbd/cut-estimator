# 签售 CUT 预估与销量监控系统 PRD

## 1. 项目背景

本产品用于根据不同签售平台后台销量、下单变化、历史 CUT、市场情绪、同行价格与平台流速特征，生成签售中选销量数据预估，帮助运营/客服/用户判断当前活动的合理购买张数。

现有竞品主要提供销量抓取、异常标记、基础 CUT 计算和表格展示，但普遍存在 UI 简陋、解释不足、决策建议弱、人工经验无法沉淀、市场价格和回归进度未纳入模型等问题。

本产品目标不是只做“销量表格”，而是做一个更专业的签售决策后台：

- 实时监控多平台、多商品销量变化
- 自动识别注水、卡单、退单、重复回传等异常
- 支持人工录入历史 CUT 和市场情绪
- 展示本次回归签售进度与上次回归对比
- 采集同行价格与市场需求信号
- 预测最后阶段销量流速
- 输出冲刺、均衡、保守三档推荐 CUT
- 给出清晰的计算依据和风险提示

## 2. 产品目标

### 2.1 核心目标

1. 降低人工盯销量、手动算 CUT 的成本。
2. 将分散在销量后台、历史记录、闲鱼/Threads、人工经验里的信息合并到同一个决策页面。
3. 让用户不仅看到“当前 CUT 是多少”，还知道“为什么是这个 CUT”。
4. 支持运营人员沉淀组合、平台、回归阶段、历史场次的经验数据。

### 2.2 成功指标

- 运营人员能在 1 个页面内完成活动判断。
- 用户能快速看到建议购买区间和风险等级。
- 异常销量能被自动标记，并允许人工复核。
- 每场活动结束后能沉淀实际 CUT，用于后续模型优化。

## 6. 核心业务流程

### 6.1 运营创建活动

1. 选择组合 / 艺人。
2. 选择签售平台。
3. 填写活动名称、商品名称、报名截止时间、公布时间。
4. 填写抽选人数、地区、报名链接、商品链接。
5. 配置抓取方式和刷新频率。
6. 绑定历史活动或上次回归数据。
7. 发布给指定用户或授权码。

### 6.2 系统抓取销量

1. 按平台规则定时抓取销量。
2. 记录每次总销量、增量、库存、金额、渠道、时间。
3. 只有销量变化时写入变化记录。
4. 自动识别异常记录并打标签。
5. 更新有效销量、理论 CUT、推荐 CUT。

### 6.3 运营复核

1. 查看异常标记。
2. 手动勾选排除或恢复某条记录。
3. 填写人工预估信息。
4. 查看本次回归进度和同行价格。
5. 调整模型权重或采用默认权重。
6. 生成最终推荐。

### 6.4 用户查看建议

1. 登录或输入激活码。
2. 查看被授权活动。
3. 进入活动详情。
4. 查看冲刺 / 均衡 / 保守三档 CUT。
5. 输入已购买数量。
6. 系统输出建议追加数量和风险说明。

### 7.3 异常检测

#### 7.3.1 异常类型

- 退单回退：正增量后出现对应负增量。
- 无效订单：正负抵消，对实际销量无影响。
- 疑似卡单：平台允许未付款计销量，后续可能回退。
- 疑似注水：平台或外部异常导致销量虚高。
- 异常大单：短时间内出现远超正常分布的增量。
- 平台重复回传：5 分钟内出现相同或高度相似记录。
- 数据延迟：长时间无数据更新但平台仍在售。
- 抓取失败：接口或页面获取失败。

#### 7.3.2 颜色建议

- 红色：高风险异常，如注水、卡单、异常大单。
- 橙色：正负抵消、疑似无效订单。
- 紫色：平台重复回传 bug。
- 黄色：待复核异常。
- 灰色：已人工排除。
- 绿色：已复核正常。

#### 7.3.3 人工标注

运营可对每条数据执行：

- 纳入计算
- 排除计算
- 标记为注水
- 标记为卡单
- 标记为退单
- 标记为重复回传
- 添加备注

所有人工操作需要记录操作人和操作时间。

### 7.4 CUT 基础计算

#### 7.4.1 基础指标

- 总销量
- 注水量
- 去水后销量
- 退单量
- 有效销量
- 抽选人数
- 理论 CUT = 有效销量 / 抽选人数
- MAX 参考值
- 异常销量占比

#### 7.4.2 用户追加建议

用户输入：

- 已购买数量
- 目标策略：冲刺 / 均衡 / 保守
- 预算上限，可选
- 可接受风险，可选

系统输出：

- 建议追加数量
- 预计总购买数量
- 风险等级
- 原因说明

### 7.5 因子 1：人工预估信息

新增“人工预估”栏目，用于录入运营经验和市场情绪。

#### 7.5.1 字段

- 近次回归 CUT
- 上次同平台 CUT
- 上次同成员 CUT
- 本次预计参与人数
- 市场情绪：冷 / 正常 / 热 / 爆
- 组合热度说明
- 成员热度说明
- 平台特殊说明
- 版本 / 特典吸引力
- 人工建议 CUT
- 人工备注

#### 7.5.2 用途

人工预估作为模型修正因子，用于补足实时销量无法反映的市场情绪、人气变化、成员差异和活动吸引力。

### 7.6 因子 2：回归签售进度

新增“回归进度”板块，用于判断当前签售位于本次回归的什么阶段。

#### 7.6.1 展示方式

采用左右两列对照：

- 左列：本次回归已发签售
- 右列：上次回归签售节奏

每列展示：

- 第几场
- 日期
- 平台
- 活动名称
- 抽选人数
- 实际 CUT
- 阶段标签：前期 / 中期 / 末场
- 是否特殊场：首场 / 末场 / 线下 / 稀缺特典

#### 7.6.2 业务规则

- 前几场通常价格更高，模型可上调。
- 中间场竞争通常较低，模型可下调。
- 末场通常更贵，模型可上调。
- 若本次回归签售数量明显少于上次，模型可上调。
- 若本次回归签售密度明显高于上次，模型可下调。

### 7.7 因子 3：同行价格

新增“市场价格监控”模块，收集闲鱼、Threads 等平台的名额报价和需求信号。

#### 7.7.1 字段

- 来源平台：闲鱼 / Threads / 其他
- 签售名称
- 成员 / 团体
- 报价
- 币种
- 想要人数 / 收藏 / 点赞 / 评论
- 发布时间
- 链接
- 卖家 / 发帖人
- 是否重复报价
- 是否疑似虚高
- 备注

#### 7.7.2 展示

- 价格区间
- 中位价
- 最高价
- 最低价
- 有效报价数量
- 想要人数总量
- 市场热度等级
- 价格趋势

#### 7.7.3 用途

同行价格作为外部市场因子，影响最终 CUT：

- 报价高且想要人数多，说明需求强，模型上调。
- 报价低且互动弱，说明需求弱，模型下调。
- 疑似虚高或重复报价不参与主计算，只作为参考。

### 7.8 因子 4：智能流速预估

新增“尾段流速预测”模块，用于预测最后一小时合理销量增长。

#### 7.8.1 输入

- 当前销量曲线
- 最后一小时剩余时间
- 当前平台历史流速
- 该组合历史签售流速
- 同平台同类型活动历史数据
- 往期 CUT 记录
- 已识别异常大单
- 当前市场热度

#### 7.8.2 输出

- 最后一小时合理新增销量区间
- 预计最终有效销量
- 乐观最终 CUT
- 均衡最终 CUT
- 保守最终 CUT
- 流速异常提醒

#### 7.8.3 排除规则

以下数据不直接用于流速预测：

- 明显异常大单
- 已标记注水
- 已标记卡单
- 已标记重复回传
- 正负抵消数据
- 人工排除数据

### 7.9 最终 CUT 模型

#### 7.9.1 推荐公式

最终推荐 CUT 由以下部分组成：

```text
最终推荐 CUT =
基础实时 CUT
+ 人工预估修正
+ 回归进度修正
+ 同行价格修正
+ 尾段流速修正
+ 平台风险修正
```

#### 7.9.2 三档输出

- 冲刺 CUT：适合预算紧、能接受较高风险。
- 均衡 CUT：系统主推荐。
- 保守 CUT：适合希望稳进。

#### 7.9.3 解释要求

每次输出必须给出原因说明，例如：

```text
当前为本次回归第 2 场，属于前期场次，模型上调 8%。
同行报价中位数偏高，想要人数较多，模型上调 6%。
该平台最后一小时历史流速偏快，模型上调 10%。
已剔除疑似卡单 160 张和重复回传 24 张。
建议均衡 CUT 为 145 张，保守 CUT 为 168 张。
```

## 12. 风险与注意事项

- 平台销量数据可能存在延迟、卡单、注水、重复回传。
- 自动异常判断只能作为参考，不能代替人工复核。

- 最终 CUT 应以区间和风险等级展示，避免承诺必中。
- 需要记录每次模型输出依据，方便复盘。
